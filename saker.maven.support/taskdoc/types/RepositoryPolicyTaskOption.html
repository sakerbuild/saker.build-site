RepositoryPolicyTaskOption
In bundle: saker.maven.support
Represents an artifact policy configuration for Maven operations. The configurations can be defined for release and snapshot artifacts separately in the repository configurations. The configuration corresponds to the <releases/> or <snapshots/> elements in a pom.xml. The option accepts a simple boolean false as its input to disable the artifact policy.

Fields
ChecksumPolicy
ChecksumPolicy
Specifies the checksum policy that should be used when incorrect checksums of artifacts are detected. May be either one of: fail, warn, ignore. The default is warn.
Enabled
Boolean
Specifies whether or not the use of the associated artifacts are enabled. If not specified, defaults to true.
UpdatePolicy
UpdatePolicy
Specifies how often the update checks of artifacts should happen from remote repositories. May be either one of: always, daily, never or interval in the format of interval:<num-minutes>. The default is daily.
